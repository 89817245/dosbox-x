00369  * but, their code also assumes that calling into DOS or the BIOS will unmask the IRQ.
00370  *

00372  *
00373  *    Pseudocode (early in demo init):

00375  *             in     al,21h

01873 //
01874 // I don't know exactly how things differ in NEC's PC-98 MS-DOS, but,
01875 // according to some strange code in Touhou Project that's responsible
01876 // for blanking the text layer, there's a "row count" variable at 0x70:0x12
01877 // that holds (number of rows - 1). Leaving that byte value at zero prevents
01878 // the game from clearing the screen (which also exposes the tile data and
01879 // overdraw of the graphics layer). A value of zero instead just causes the
01880 // first text character row to be filled in, not the whole visible text layer.
01881 //
01882 // Pseudocode of the routine:
01883 //
01884 // XOR AX,AX
01885 // MOV ES,AX
01886 // MOV AL,ES:[0712h]            ; AX = BYTE [0x70:0x12] zero extend (ex. 0x18 == 24)
01887 // INC AX                       ; AX++                              (ex. becomes 0x19 == 25)
01888 // MOV DX,AX
01889 // SHL DX,1
01890 // SHL DX,1                     ; DX *= 4
01891 // ADD DX,AX                    ; DX += AX     equiv. DX = AX * 5
01892 // MOV CL,4h
01893 // SHL DX,CL                    ; DX <<= 4     equiv. DX = AX * 0x50  or  DX = AX * 80
01894 // ...
01895 // MOV AX,0A200h
01896 // MOV ES,AX
01897 // MOV AX,(solid black overlay block attribute)
01898 // MOV CX,DX
01899 // REP STOSW
01900 //
01901 // When the routine is done, the graphics layer is obscured by text character cells that
01902 // represent all black (filled in) so that the game can later "punch out" the regions
01903 // of the graphics layer it wants you to see. TH02 relies on this as well to flash the
01904 // screen and open from the center to show the title screen. During gameplay, the text
01905 // layer is used to obscure sprite overdraw when a sprite is partially off-screen as well
01906 // as hidden tile data on the right hand half of the screen that the game read/write
01907 // copies through the GDC pattern/tile registers to make the background. When the text
01908 // layer is not present it's immediately apparent that the sprite renderer makes no attempt
01909 // to clip sprites within the screen, but instead relies on the text overlay to hide the
01910 // overdraw.
01911 //
01912 // this means that on PC-98 one of two things are true. either:
01913 //  - NEC's variation of MS-DOS loads the base kernel higher up (perhaps at 0x80:0x00?)
01914 //    and the BIOS data area lies from 0x40:00 to 0x7F:00
01915 //
01916 //    or
01917 //
01918 //  - NEC's variation loads at 0x70:0x00 (same as IBM PC MS-DOS) and Touhou Project
