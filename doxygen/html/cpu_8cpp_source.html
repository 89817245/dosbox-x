00081 /* [cpu] setting realbig16.
00082  * If set, allow code to switch back to real mode with the B (big) set in the
00083  * code selector, and retain the state of the B bit while running in 16-bit
00084  * real mode. Needed for demos like Project Angel.
00085  *
00086  * Modifications are a derivative of this patch:
00087  *
00088  * cpu.diff from http://www.vogons.org/viewtopic.php?f=33&t=28226&start=5
00089  *
00090  * The main difference between that patch and the modifications derived is that
00091  * I modified additional points to keep the big bit set (the original cpu.diff
00092  * missed the CALL and JMP emulation that also reset the flag)
00093  *
00094  * It's well known that DOS programs can access all 4GB of addressable memory by
00095  * jumping into protected mode, loading segment registers with a 4GB limit, then
00096  * jumping back to real mode without reloading the segment registers, knowing
00097  * that Intel processors will not update the shadow part of the segment register
00098  * in real mode. I'm guessing that what Project Angel is doing, is using the same
00099  * abuse of protected mode to also set the B (big) bit in the code segment so that
00100  * it's code segment can extend past 64KB (huge unreal mode), which works until
00101  * something like an interrupt chops off the top 16 bits of the instruction pointer.
00102  *
00103  * I want to clarify that realbig16 is an OPTION that is off by default, because
00104  * I am uncertain at this time whether or not the patch breaks any DOS games or
00105  * OS emulation. It is rare for a DOS game or demo to actually abuse the CPU in
00106  * that way, so it is set up that you have to enable it if you need it. --J.C.
00107  *
00108  * J.C. TODO: Write a program that abuses the B (big) bit in real mode in the same
00109  *            way that Project Angel supposedly does, see if it works, then test it
00110  *            and Project Angel on some old 386/486/Pentium systems lying around to
00111  *            see how compatible such abuse is with PC hardware. That would make a
00112  *            good Hackipedia.org episode as well. --J.C.
00113  *
00114  * 2014/01/19: I can attest that this patch does indeed allow Project Angel to
00115  *             run when realbig16=true. And if GUS emulation is active, there is
00116  *             music as well. Now as for reliability... testing shows that one of
00117  *             three things can happen when you run the demo:
00118  *
00119  *             1) the demo hangs on startup, either right away or after it starts
00120  *                the ominous music (if you sit for 30 seconds waiting for the
00121  *                music to build up and nothing happens, consider closing the
00122  *                emulator and trying again).
00123  *
00124  *             2) the demo runs perfectly fine, but timing is slightly screwed up,
00125  *                and parts of the music sound badly out of sync with each other,
00126  *                or randomly slows to about 1/2 speed in some sections, animations
00127  *                run slow sometimes. If this happens, make sure you didn't set
00128  *                forcerate=ntsc.
00129  *
00130  *             3) the demo runs perfectly fine, with no timing issues, except that
00131  *                DOSBox's S3 emulation is not quite on-time and the bottom 1/4th
00132  *                of the screen flickers with the contents of the next frame that
00133  *                the demo is still drawing :(
00134  *
00135  *             --J.C. */

00173 /* NTS: From the Intel 80386 programmer's reference manual:
00174  *
00175  * "
00176  *   9.2.1 NMI Masks Further NMIs
00177  *
00178  *   While an NMI handler is executing, the processor ignores further interrupt
00179  *   signals at the NMI pin until the next IRET instruction is executed.
00180  * "
00181  *
00182  * This is why, further down, CPU_IRET() clears the CPU_NMI_active flag.
00183  *
00184  *
00185  * And, in response to my father's incredulous response regarding the fact that
00186  * NMI is edge-triggered (from the Intel 386SX Microprocessor datasheet):
00187  *
00188  * "
00189  *   Non-Maskable Interrupt Request (NMI))
00190  *
00191  *   This input indicates a request for interrupt service
00192  *   which cannot be masked by software. The non-
00193  *   maskable interrupt request is always processed ac-
00194  *   cording to the pointer or gate in slot 2 of the interrupt
00195  *   table. Because of the fixed NMI slot assignment, no
00196  *   interrupt acknowledge cycles are performed when
00197  *   processing NMI.
00198  *
00199  *   NMI is an active HIGH, rising edge-sensitive asyn-
00200  *   chronous signal. Setup and hold times, t27 and and t28,
00201  *   relative to the CLK2 signal must be met to guarantee
00202  *   recognition at a particular clock edge. To assure rec-
00203  *   ognition of NMI, it must be inactive for at least eight
00204  *   CLK2 periods, and then be active for at least eight
00205  *   CLK2 periods before the beginning of the instruction
00206  *   boundary in the Intel386 SX Microprocessor's Exe-
00207  *   cution Unit.
00208  *
00209  *   Once NMI processing has begun, no additional
00210  *   NMI's are processed until after the next IRET in-
00211  *   struction, which is typically the end of the NMI serv-
00212  *   ice routine. If NMI is re-asserted prior to that time,
00213  *   however, one rising edge on NMI will be remem-
00214  *   bered for processing after executing the next IRET
00215  *   instruction
00216  * "
00217  *
00218  * From the Pentium Pro Processor datasheet:
00219  *
00220  * "
00221  *   A.38 NMI (I)
00222  *
00223  *   The NMI signal is the Non-maskable Interrupt signal.
00224  *   It is the state of the LINT1 signal when APIC is
00225  *   disabled. Asserting NMI causes an interrupt with an
00226  *   internally supplied vector value of 2. An external
00227  *   interrupt-acknowledge transaction is not generated. If
00228  *   NMI is asserted during the execution of an NMI
00229  *   service routine, it remains pending and is recognized
00230  *   after the IRET is executed by the NMI service
00231  *   routine. At most, one assertion of NMI is held
00232  *   pending.
00233  *
00234  *   NMI is rising-edge sensitive. Recognition of NMI is
00235  *   guaranteed in a specific clock if it is asserted
00236  *   synchronously and meets the setup and hold times. If
00237  *   asserted asynchronously, active and inactive pulse
00238  *   widths must be a minimum of two clocks. In FRC
00239  *   mode, NMI must be synchronous to BCLK.
00240  * "
00241  *
00242  * Similar references exist in the Pentium III and Pentium 4
00243  * datasheets, while later on in the Core 2 datasheets there
00244  * is no mention whatsoever to the NMI that I can find.
00245  */

00361 void CPU_SetFlags(Bitu word,Bitu mask) {
00362         /* 8086/286 flags manipulation.
00363          * For more information read about the Intel CPU detection algorithm and other bits of info:
00364          * [http://www.rcollins.org/ddj/Sep96/Sep96.html] */
00365 
00366         /* 8086: bits 12-15 cannot be zeroed */
00367         if (CPU_ArchitectureType <= CPU_ARCHTYPE_80186) {
00368                 /* update mask and word to ensure bits 12-15 are set */
00369                 word |= 0xF000;
00370                 mask |= 0xF000;
00371         }
00372         /* 286 real mode: bits 12-15 bits cannot be set, always zero */
00373         else if (CPU_ArchitectureType <= CPU_ARCHTYPE_286) {
00374                 if (!(cpu.cr0 & CR0_PROTECTION)) {
00375                         /* update mask and word to ensure bits 12-15 are zero */
00376                         word &= ~0xF000;
00377                         mask |= 0xF000;
00378                 }
00379         }
00380         else {
00381                 mask |= CPU_extflags_toggle;    // ID-flag and AC-flag can be toggled on CPUID-supporting CPUs
00382         }
00383 
00384         reg_flags=(reg_flags & ~mask)|(word & mask)|2;
00385         cpu.direction=1-((reg_flags & FLAG_DF) >> 9);
00386         // ^ NTS: Notice the DF flag is bit 10. This code computes (reg_flags & FLAG_DF) >> 9 on purpose.
00387         //        It's not a typo (9 vs 10), it's done to set cpu.direction to either 1 or -1.
00388 }

00424                 CPU_SetFlags(CPU_Pop32(),mask);
00425         else CPU_SetFlags(CPU_Pop16(),mask & 0xffff);
00426         DestroyConditionFlags();
00427         return false;

01015                                                 EXCEPTION_NP,(gate_sel & 0xfffc)+(type&CPU_INT_SOFTWARE)?0:1)
01016                                         if ((reg_flags & FLAG_VM) && (cs_dpl<cpu.cpl))
01017                                                 E_Exit("V86 interrupt doesn't change to pl0");  // or #GP(cs_sel)
01018 
01019                                         // commit point
01020 do_interrupt:
01021                                         if (gate.Type() & 0x8) {        /* 32-bit Gate */
01022                                                 CPU_Push32(reg_flags);
01023                                                 CPU_Push32(SegValue(cs));
01024                                                 CPU_Push32(oldeip);
01025                                                 if (type & CPU_INT_HAS_ERROR) CPU_Push32(cpu.exception.error);
01026                                         } else {                                        /* 16-bit gate */
01027                                                 CPU_Push16(reg_flags & 0xffff);
01028                                                 CPU_Push16(SegValue(cs));
01029                                                 CPU_Push16(oldeip);
01030                                                 if (type & CPU_INT_HAS_ERROR) CPU_Push16(cpu.exception.error);
01031                                         }
01032                                         break;          
01033                                 default:
01034                                         E_Exit("INT:Gate Selector points to illegal descriptor with type %x",(int)cs_desc.Type());
01035                                 }
01036 
01037                                 Segs.val[cs]=(gate_sel&0xfffc) | cpu.cpl;
01038                                 Segs.phys[cs]=cs_desc.GetBase();
01039                                 Segs.limit[cs]=do_seg_limits?cs_desc.GetLimit():(~0UL);
01040                                 Segs.expanddown[cs]=cs_desc.GetExpandDown();
01041                                 cpu.code.big=cs_desc.Big()>0;
01042                                 reg_eip=gate_off;
01043 
01044                                 if (!(gate.Type()&1)) {
01045                                         SETFLAGBIT(IF,false);
01046                                 }
01047                                 SETFLAGBIT(TF,false);
01048                                 SETFLAGBIT(NT,false);
01049                                 SETFLAGBIT(VM,false);
01050                                 LOG(LOG_CPU,LOG_NORMAL)("INT:Gate to %X:%X big %d %s",gate_sel,gate_off,cs_desc.Big(),gate.Type() & 0x8 ? "386" : "286");
01051                                 return;
01052                         }
01053                 case DESC_TASK_GATE:
01054                         CPU_CHECK_COND(!gate.saved.seg.p,
01055                                 "INT:Gate segment not present",
01056                                 EXCEPTION_NP,num*8+2+(type&CPU_INT_SOFTWARE)?0:1)
01057 
01058                         CPU_SwitchTask(gate.GetSelector(),TSwitch_CALL_INT,oldeip);
01059                         if (type & CPU_INT_HAS_ERROR) {
01060                                 //TODO Be sure about this, seems somewhat unclear
01061                                 if (cpu_tss.is386) CPU_Push32(cpu.exception.error);
01062                                 else CPU_Push16(cpu.exception.error);
01063                         }
01064                         return;
01065                 default:
01066                         E_Exit("Illegal descriptor type %X for int %X",(int)gate.Type(),(int)num);
01067                 }
01068                 }
01069                 catch (GuestPageFaultException &pf) {
01070                         LOG_MSG("CPU_Interrupt() interrupted");
01071                         CPU_SetSegGeneral(ss,old_ss);
01072                         reg_esp = old_esp;
01073                         CPU_SetCPL(old_cpl);
01074                         throw;
01075                 }
01076         }
01077         assert(1);
01078         return ; // make compiler happy
01079 }

01085         /* x86 CPUs consider IRET the completion of an NMI, no matter where it happens */

01096                         CPU_Exception_Level[which]--;

01107                         reg_eip=CPU_Pop16();

01140                                         reg_esp=(tempesp&cpu.stack.notmask)|((tempesp+2)&cpu.stack.mask);

01142                                         reg_eip=(Bit32u)new_eip;
01143                                         SegSet16(cs,new_cs);
01144                                         /* IOPL can not be modified in v86 mode by IRET */
01145                                         CPU_SetFlags(new_flags,FMASK_NORMAL|FLAG_NT);
01146                                 }
01147                                 }
01148                                 catch (GuestPageFaultException &pf) {

01151                                         throw;

01154                                 DestroyConditionFlags();

01156                         }
01157                 }

01159                 if (GETFLAG(NT)) {
01160                         if (GETFLAG(VM)) E_Exit("Pmode IRET with VM bit set");

01197

01199                                 CPU_SetSegGeneral(es,n_es);
01200                                 CPU_SetSegGeneral(ds,n_ds);
01201                                 CPU_SetSegGeneral(fs,n_fs);
01202                                 CPU_SetSegGeneral(gs,n_gs);
01203                                 reg_esp=n_esp;

01206                                 LOG(LOG_CPU,LOG_NORMAL)("IRET:Back to V86: CS:%X IP %X SS:%X SP %X FLAGS:%X",SegValue(cs),reg_eip,SegValue(ss),reg_esp,reg_flags);      
01207                                 return;
01208                                 }
01209                                 catch (GuestPageFaultException &pf) {
01210                                         LOG_MSG("CPU_IRET() interrupted prot use32");
01211                                         reg_esp = orig_esp;
01212                                         throw;
01213                                 }
01214                         }
01215                         if (n_flags & FLAG_VM) E_Exit("IRET from pmode to v86 with CPL!=0");
01216                 } else {
01217                         n_eip=mem_readw(SegPhys(ss) + (reg_esp & cpu.stack.mask));
01218                         tempesp=(reg_esp&cpu.stack.notmask)|((reg_esp+2)&cpu.stack.mask);
01219                         n_cs_sel=mem_readw(SegPhys(ss) + (tempesp & cpu.stack.mask));
01220                         tempesp=(tempesp&cpu.stack.notmask)|((tempesp+2)&cpu.stack.mask);
01221                         n_flags=mem_readw(SegPhys(ss) + (tempesp & cpu.stack.mask));
01222                         n_flags|=(reg_flags & 0xffff0000);
01223                         tempesp=(tempesp&cpu.stack.notmask)|((tempesp+2)&cpu.stack.mask);
01224 
01225                         if (n_flags & FLAG_VM) E_Exit("VM Flag in 16-bit iret");
01226                 }
01227                 CPU_CHECK_COND((n_cs_sel & 0xfffc)==0,
01228                         "IRET:CS selector zero",
01229                         EXCEPTION_GP,0)
01230                 Bitu n_cs_rpl=n_cs_sel & 3;
01231                 Descriptor n_cs_desc;
01232                 CPU_CHECK_COND(!cpu.gdt.GetDescriptor(n_cs_sel,n_cs_desc),
01233                         "IRET:CS selector beyond limits",
01234                         EXCEPTION_GP,n_cs_sel & 0xfffc)

01236                         "IRET to lower privilege",
01237                         EXCEPTION_GP,n_cs_sel & 0xfffc)
01238 
01239                 switch (n_cs_desc.Type()) {
01240                 case DESC_CODE_N_NC_A:  case DESC_CODE_N_NC_NA:
01241                 case DESC_CODE_R_NC_A:  case DESC_CODE_R_NC_NA:

01244                                 EXCEPTION_GP,n_cs_sel & 0xfffc)

01246                 case DESC_CODE_N_C_A:   case DESC_CODE_N_C_NA:

01250                                 EXCEPTION_GP,n_cs_sel & 0xfffc)
01251                         break;
01252                 default:
01253                         E_Exit("IRET:Illegal descriptor type %X",(int)n_cs_desc.Type());
01254                 }

01256                         "IRET with nonpresent code segment",
01257                         EXCEPTION_NP,n_cs_sel & 0xfffc)
01258

01260                         /* Return to same level */
01261 
01262                         // commit point
01263                         reg_esp=tempesp;
01264                         Segs.expanddown[cs]=n_cs_desc.GetExpandDown();
01265                         Segs.limit[cs]=do_seg_limits?n_cs_desc.GetLimit():(~0UL);
01266                         Segs.phys[cs]=n_cs_desc.GetBase();

01280                                 n_esp=mem_readd(SegPhys(ss) + (tempesp & cpu.stack.mask));

01315

01321

01323                         if (GETFLAG_IOPL<cpu.cpl) mask &= (~FLAG_IF);
01324                         CPU_SetFlags(n_flags,mask);
01325                         DestroyConditionFlags();

01331                         Segs.phys[ss]=n_ss_desc.GetBase();

01335                                 cpu.stack.big=true;
01336                                 cpu.stack.mask=0xffffffff;
01337                                 cpu.stack.notmask=0;
01338                                 reg_esp=n_esp;
01339                         } else {
01340                                 cpu.stack.big=false;
01341                                 cpu.stack.mask=0xffff;
01342                                 cpu.stack.notmask=0xffff0000;

01357         if (!cpu.pmode || (reg_flags & FLAG_VM)) {
01358                 if (!use32) {
01359                         reg_eip=offset&0xffff;
01360                 } else {
01361                         reg_eip=offset;
01362                 }

01377                 case DESC_CODE_R_NC_A:          case DESC_CODE_R_NC_NA:

01390                                 "JMP:C:CPL < DPL",
01391                                 EXCEPTION_GP,selector & 0xfffc)

01393                         if (!desc.saved.seg.p) {
01394                                 // win
01395                                 CPU_Exception(EXCEPTION_NP,selector & 0xfffc);

01397                         }
01398

01405                         reg_eip=offset;

01410                                 EXCEPTION_GP,selector & 0xfffc)
01411                         CPU_CHECK_COND(desc.DPL()<rpl,
01412                                 "JMP:TSS:dpl<rpl",
01413                                 EXCEPTION_GP,selector & 0xfffc)
01414                         LOG(LOG_CPU,LOG_NORMAL)("JMP:TSS to %X",selector);
01415                         CPU_SwitchTask(selector,TSwitch_JMP,oldeip);

01417                 default:
01418                         E_Exit("JMP Illegal descriptor type %X",(int)desc.Type());
01419                 }
01420         }
01421         assert(1);

01471                 case DESC_CODE_R_C_A:case DESC_CODE_R_C_NA:

01473                                 "CALL:CODE:C:DPL>CPL",

01701         Bitu orig_esp = reg_esp;
01702 
01703         if (!cpu.pmode || (reg_flags & FLAG_VM)) {
01704                 try {
01705                 Bitu new_ip,new_cs;
01706                 if (!use32) {
01707                         new_ip=CPU_Pop16();
01708                         new_cs=CPU_Pop16();
01709                 } else {
01710                         new_ip=CPU_Pop32();
01711                         new_cs=CPU_Pop32() & 0xffff;
01712                 }

01718                 }
01719                 catch (GuestPageFaultException &pf) {
01720                         LOG_MSG("CPU_RET() interrupted real/vm86");
01721                         reg_esp = orig_esp;
01722                         throw;
01723                 }
01724         } else {
01725                 Bitu offset,selector;
01726                 if (!use32) selector    = mem_readw(SegPhys(ss) + (reg_esp & cpu.stack.mask) + 2);
01727                 else            selector        = mem_readd(SegPhys(ss) + (reg_esp & cpu.stack.mask) + 4) & 0xffff;
01728 
01729                 Descriptor desc;
01730                 Bitu rpl=selector & 3;
01731                 if(rpl < cpu.cpl) {
01732                         // win setup
01733                         CPU_Exception(EXCEPTION_GP,selector & 0xfffc);
01734                         return;
01735                 }
01736 
01737                 CPU_CHECK_COND((selector & 0xfffc)==0,
01738                         "RET:CS selector zero",
01739                         EXCEPTION_GP,0)

01741                         "RET:CS beyond limits",
01742                         EXCEPTION_GP,selector & 0xfffc)
01743

01745                         /* Return to same level */
01746                         switch (desc.Type()) {
01747                         case DESC_CODE_N_NC_A:case DESC_CODE_N_NC_NA:
01748                         case DESC_CODE_R_NC_A:case DESC_CODE_R_NC_NA:
01749                                 CPU_CHECK_COND(cpu.cpl!=desc.DPL(),
01750                                         "RET to NC segment of other privilege",
01751                                         EXCEPTION_GP,selector & 0xfffc)

01753                         case DESC_CODE_N_C_A:case DESC_CODE_N_C_NA:
01754                         case DESC_CODE_R_C_A:case DESC_CODE_R_C_NA:
01755                                 CPU_CHECK_COND(desc.DPL()>cpu.cpl,
01756                                         "RET to C segment of higher privilege",
01757                                         EXCEPTION_GP,selector & 0xfffc)
01758                                 break;
01759                         default:
01760                                 E_Exit("RET from illegal descriptor type %X",(int)desc.Type());
01761                         }
01762 RET_same_level:
01763                         if (!desc.saved.seg.p) {
01764                                 // borland extender (RTM)
01765                                 CPU_Exception(EXCEPTION_NP,selector & 0xfffc);
01766                                 return;
01767                         }
01768 
01769                         // commit point
01770                         try {
01771                         if (!use32) {
01772                                 offset=CPU_Pop16();
01773                                 selector=CPU_Pop16();
01774                         } else {
01775                                 offset=CPU_Pop32();
01776                                 selector=CPU_Pop32() & 0xffff;
01777                         }
01778                         }
01779                         catch (GuestPageFaultException &pf) {
01780                                 LOG_MSG("CPU_RET() interrupted prot rpl==cpl");
01781                                 reg_esp = orig_esp;
01782                                 throw;
01783                         }
01784 
01785                         Segs.expanddown[cs]=desc.GetExpandDown();
01786                         Segs.limit[cs]=do_seg_limits?desc.GetLimit():(~0UL);
01787                         Segs.phys[cs]=desc.GetBase();

01789                         Segs.val[cs]=selector;
01790                         reg_eip=offset;
01791                         if (cpu.stack.big) {
01792                                 reg_esp+=bytes;
01793                         } else {
01794                                 reg_sp+=bytes;
01795                         }

01800                         switch (desc.Type()) {
01801                         case DESC_CODE_N_NC_A:case DESC_CODE_N_NC_NA:

01803                                 CPU_CHECK_COND(desc.DPL()!=rpl,
01804                                         "RET to outer NC segment with DPL!=RPL",
01805                                         EXCEPTION_GP,selector & 0xfffc)
01806                                 break;
01807                         case DESC_CODE_N_C_A:case DESC_CODE_N_C_NA:
01808                         case DESC_CODE_R_C_A:case DESC_CODE_R_C_NA:
01809                                 CPU_CHECK_COND(desc.DPL()>rpl,

01815                         }

01818                                 "RET:Outer level:CS not present",
01819                                 EXCEPTION_NP,selector & 0xfffc)
01820

01846                                 EXCEPTION_GP,0)

01849                         CPU_CHECK_COND(!cpu.gdt.GetDescriptor(n_ss,n_ss_desc),
01850                                 "RET:SS beyond limits",
01851                                 EXCEPTION_GP,n_ss & 0xfffc)

01859                                 break;

01868                         Segs.expanddown[cs]=desc.GetExpandDown();

01978 /* On shutdown, DOSBox needs to snap back to real mode
01979  * so that it's shutdown code doesn't cause page faults
01980  * trying to clean up DOS structures when we've booted
01981  * a 32-bit OS. It shouldn't be cleaning up DOS structures
01982  * anyway in that case considering they're likely obliterated
01983  * by the guest OS, but that's something we'll clean up
01984  * later. */

01986     if (snap_cpu_snapped) return;

01988     SETFLAGBIT(IF,false);       /* forcibly clear interrupt flag */
01989 
01990     cpu.code.big = false;   /* force back to 16-bit */
01991     cpu.stack.big = false;
01992     cpu.stack.mask = 0xffff;
01993     cpu.stack.notmask = 0xffff0000;
01994 
01995     snap_cpu_saved_cr0 = cpu.cr0;
01996     snap_cpu_saved_cr2 = paging.cr2;
01997     snap_cpu_saved_cr3 = paging.cr3;
01998 
01999     CPU_SET_CRX(0,0);   /* force CPU to real mode */
02000     CPU_SET_CRX(2,0);   /* disable paging */
02001     CPU_SET_CRX(3,0);   /* clear the page table dir */
02002 
02003     cpu.idt.SetBase(0);         /* or ELSE weird things will happen when INTerrupts are run */
02004     cpu.idt.SetLimit(1023);
02005 
02006     snap_cpu_snapped = true;

02504 bool CPU_CPUID(void) {
02505         if (CPU_ArchitectureType < CPU_ARCHTYPE_486NEW) return false;
02506         switch (reg_eax) {
02507         case 0: /* Vendor ID String and maximum level? */
02508                 reg_eax=1;  /* Maximum level */ 
02509                 reg_ebx='G' | ('e' << 8) | ('n' << 16) | ('u'<< 24); 
02510                 reg_edx='i' | ('n' << 8) | ('e' << 16) | ('I'<< 24); 
02511                 reg_ecx='n' | ('t' << 8) | ('e' << 16) | ('l'<< 24); 
02512                 break;
02513         case 1: /* get processor type/family/model/stepping and feature flags */
02514                 if ((CPU_ArchitectureType == CPU_ARCHTYPE_486NEW) ||
02515                         (CPU_ArchitectureType == CPU_ARCHTYPE_MIXED)) {
02516                         reg_eax=0x402;          /* intel 486dx */
02517                         reg_ebx=0;                      /* Not Supported */
02518                         reg_ecx=0;                      /* No features */
02519                         reg_edx=enable_fpu?1:0; /* FPU */
02520                 } else if (CPU_ArchitectureType == CPU_ARCHTYPE_PENTIUM) {
02521                         reg_eax=0x513;          /* intel pentium */
02522                         reg_ebx=0;                      /* Not Supported */
02523                         reg_ecx=0;                      /* No features */
02524                         reg_edx=0x00000010|(enable_fpu?1:0);    /* FPU+TimeStamp/RDTSC */
02525                         if (enable_msr) reg_edx |= 0x20; /* ModelSpecific/MSR */
02526             if (enable_cmpxchg8b) reg_edx |= 0x100; /* CMPXCHG8B */
02527                 } else if (CPU_ArchitectureType == CPU_ARCHTYPE_P55CSLOW) {
02528                         reg_eax=0x543;          /* intel pentium mmx (P55C) */
02529                         reg_ebx=0;                      /* Not Supported */
02530                         reg_ecx=0;                      /* No features */
02531                         reg_edx=0x00800010|(enable_fpu?1:0);    /* FPU+TimeStamp/RDTSC+MMX+ModelSpecific/MSR */
02532                         if (enable_msr) reg_edx |= 0x20; /* ModelSpecific/MSR */
02533             if (enable_cmpxchg8b) reg_edx |= 0x100; /* CMPXCHG8B */
02534                 } else if (CPU_ArchitectureType == CPU_ARCHTYPE_PPROSLOW) {
02535                         reg_eax=0x612;          /* intel pentium pro */
02536                         reg_ebx=0;                      /* Not Supported */
02537                         reg_ecx=0;                      /* No features */
02538                         reg_edx=0x00008011;     /* FPU+TimeStamp/RDTSC */

02546                 reg_ebx=0;
02547                 reg_ecx=0;
