00083 /* [cpu] setting realbig16.
00084  * If set, allow code to switch back to real mode with the B (big) set in the
00085  * code selector, and retain the state of the B bit while running in 16-bit
00086  * real mode. Needed for demos like Project Angel.
00087  *
00088  * Modifications are a derivative of this patch:
00089  *
00090  * cpu.diff from http://www.vogons.org/viewtopic.php?f=33&t=28226&start=5
00091  *
00092  * The main difference between that patch and the modifications derived is that
00093  * I modified additional points to keep the big bit set (the original cpu.diff
00094  * missed the CALL and JMP emulation that also reset the flag)
00095  *
00096  * It's well known that DOS programs can access all 4GB of addressable memory by
00097  * jumping into protected mode, loading segment registers with a 4GB limit, then
00098  * jumping back to real mode without reloading the segment registers, knowing
00099  * that Intel processors will not update the shadow part of the segment register
00100  * in real mode. I'm guessing that what Project Angel is doing, is using the same
00101  * abuse of protected mode to also set the B (big) bit in the code segment so that
00102  * it's code segment can extend past 64KB (huge unreal mode), which works until
00103  * something like an interrupt chops off the top 16 bits of the instruction pointer.
00104  *
00105  * I want to clarify that realbig16 is an OPTION that is off by default, because
00106  * I am uncertain at this time whether or not the patch breaks any DOS games or
00107  * OS emulation. It is rare for a DOS game or demo to actually abuse the CPU in
00108  * that way, so it is set up that you have to enable it if you need it. --J.C.
00109  *
00110  * J.C. TODO: Write a program that abuses the B (big) bit in real mode in the same
00111  *            way that Project Angel supposedly does, see if it works, then test it
00112  *            and Project Angel on some old 386/486/Pentium systems lying around to
00113  *            see how compatible such abuse is with PC hardware. That would make a
00114  *            good Hackipedia.org episode as well. --J.C.
00115  *
00116  * 2014/01/19: I can attest that this patch does indeed allow Project Angel to
00117  *             run when realbig16=true. And if GUS emulation is active, there is
00118  *             music as well. Now as for reliability... testing shows that one of
00119  *             three things can happen when you run the demo:
00120  *
00121  *             1) the demo hangs on startup, either right away or after it starts
00122  *                the ominous music (if you sit for 30 seconds waiting for the
00123  *                music to build up and nothing happens, consider closing the
00124  *                emulator and trying again).
00125  *
00126  *             2) the demo runs perfectly fine, but timing is slightly screwed up,
00127  *                and parts of the music sound badly out of sync with each other,
00128  *                or randomly slows to about 1/2 speed in some sections, animations
00129  *                run slow sometimes. If this happens, make sure you didn't set
00130  *                forcerate=ntsc.
00131  *
00132  *             3) the demo runs perfectly fine, with no timing issues, except that
00133  *                DOSBox's S3 emulation is not quite on-time and the bottom 1/4th
00134  *                of the screen flickers with the contents of the next frame that
00135  *                the demo is still drawing :(
00136  *
00137  *             --J.C. */

00175 /* NTS: From the Intel 80386 programmer's reference manual:
00176  *
00177  * "
00178  *   9.2.1 NMI Masks Further NMIs
00179  *
00180  *   While an NMI handler is executing, the processor ignores further interrupt
00181  *   signals at the NMI pin until the next IRET instruction is executed.
00182  * "
00183  *
00184  * This is why, further down, CPU_IRET() clears the CPU_NMI_active flag.
00185  *
00186  *
00187  * And, in response to my father's incredulous response regarding the fact that
00188  * NMI is edge-triggered (from the Intel 386SX Microprocessor datasheet):
00189  *
00190  * "
00191  *   Non-Maskable Interrupt Request (NMI))
00192  *
00193  *   This input indicates a request for interrupt service
00194  *   which cannot be masked by software. The non-
00195  *   maskable interrupt request is always processed ac-
00196  *   cording to the pointer or gate in slot 2 of the interrupt
00197  *   table. Because of the fixed NMI slot assignment, no
00198  *   interrupt acknowledge cycles are performed when
00199  *   processing NMI.
00200  *
00201  *   NMI is an active HIGH, rising edge-sensitive asyn-
00202  *   chronous signal. Setup and hold times, t27 and and t28,
00203  *   relative to the CLK2 signal must be met to guarantee
00204  *   recognition at a particular clock edge. To assure rec-
00205  *   ognition of NMI, it must be inactive for at least eight
00206  *   CLK2 periods, and then be active for at least eight
00207  *   CLK2 periods before the beginning of the instruction
00208  *   boundary in the Intel386 SX Microprocessor's Exe-
00209  *   cution Unit.
00210  *
00211  *   Once NMI processing has begun, no additional
00212  *   NMI's are processed until after the next IRET in-
00213  *   struction, which is typically the end of the NMI serv-
00214  *   ice routine. If NMI is re-asserted prior to that time,
00215  *   however, one rising edge on NMI will be remem-
00216  *   bered for processing after executing the next IRET
00217  *   instruction
00218  * "
00219  *
00220  * From the Pentium Pro Processor datasheet:
00221  *
00222  * "
00223  *   A.38 NMI (I)
00224  *
00225  *   The NMI signal is the Non-maskable Interrupt signal.
00226  *   It is the state of the LINT1 signal when APIC is
00227  *   disabled. Asserting NMI causes an interrupt with an
00228  *   internally supplied vector value of 2. An external
00229  *   interrupt-acknowledge transaction is not generated. If
00230  *   NMI is asserted during the execution of an NMI
00231  *   service routine, it remains pending and is recognized
00232  *   after the IRET is executed by the NMI service
00233  *   routine. At most, one assertion of NMI is held
00234  *   pending.
00235  *
00236  *   NMI is rising-edge sensitive. Recognition of NMI is
00237  *   guaranteed in a specific clock if it is asserted
00238  *   synchronously and meets the setup and hold times. If
00239  *   asserted asynchronously, active and inactive pulse
00240  *   widths must be a minimum of two clocks. In FRC
00241  *   mode, NMI must be synchronous to BCLK.
00242  * "
00243  *
00244  * Similar references exist in the Pentium III and Pentium 4
00245  * datasheets, while later on in the Core 2 datasheets there
00246  * is no mention whatsoever to the NMI that I can find.
00247  */

00363 void CPU_SetFlags(Bitu word,Bitu mask) {
00364         /* 8086/286 flags manipulation.
00365          * For more information read about the Intel CPU detection algorithm and other bits of info:
00366          * [http://www.rcollins.org/ddj/Sep96/Sep96.html] */
00367 
00368         /* 8086: bits 12-15 cannot be zeroed */
00369         if (CPU_ArchitectureType <= CPU_ARCHTYPE_80186) {
00370                 /* update mask and word to ensure bits 12-15 are set */
00371                 word |= 0xF000;
00372                 mask |= 0xF000;
00373         }
00374         /* 286 real mode: bits 12-15 bits cannot be set, always zero */
00375         else if (CPU_ArchitectureType <= CPU_ARCHTYPE_286) {
00376                 if (!(cpu.cr0 & CR0_PROTECTION)) {
00377                         /* update mask and word to ensure bits 12-15 are zero */
00378                         word &= ~0xF000;
00379                         mask |= 0xF000;
00380                 }
00381         }
00382         else {
00383                 mask |= CPU_extflags_toggle;    // ID-flag and AC-flag can be toggled on CPUID-supporting CPUs
00384         }
00385 
00386         reg_flags=(reg_flags & ~mask)|(word & mask)|2;
00387         cpu.direction=1-((reg_flags & FLAG_DF) >> 9);
00388         // ^ NTS: Notice the DF flag is bit 10. This code computes (reg_flags & FLAG_DF) >> 9 on purpose.
00389         //        It's not a typo (9 vs 10), it's done to set cpu.direction to either 1 or -1.
00390 }

01017                                                 EXCEPTION_NP,(gate_sel & 0xfffc)+(type&CPU_INT_SOFTWARE)?0:1)
01018                                         if ((reg_flags & FLAG_VM) && (cs_dpl<cpu.cpl))
01019                                                 E_Exit("V86 interrupt doesn't change to pl0");  // or #GP(cs_sel)
01020 
01021                                         // commit point
01022 do_interrupt:
01023                                         if (gate.Type() & 0x8) {        /* 32-bit Gate */
01024                                                 CPU_Push32(reg_flags);
01025                                                 CPU_Push32(SegValue(cs));
01026                                                 CPU_Push32(oldeip);
01027                                                 if (type & CPU_INT_HAS_ERROR) CPU_Push32(cpu.exception.error);
01028                                         } else {                                        /* 16-bit gate */
01029                                                 CPU_Push16(reg_flags & 0xffff);
01030                                                 CPU_Push16(SegValue(cs));
01031                                                 CPU_Push16(oldeip);
01032                                                 if (type & CPU_INT_HAS_ERROR) CPU_Push16(cpu.exception.error);
01033                                         }
01034                                         break;          
01035                                 default:
01036                                         E_Exit("INT:Gate Selector points to illegal descriptor with type %x",(int)cs_desc.Type());
01037                                 }
01038 
01039                                 Segs.val[cs]=(gate_sel&0xfffc) | cpu.cpl;
01040                                 Segs.phys[cs]=cs_desc.GetBase();
01041                                 Segs.limit[cs]=do_seg_limits?cs_desc.GetLimit():(~0UL);
01042                                 Segs.expanddown[cs]=cs_desc.GetExpandDown();
01043                                 cpu.code.big=cs_desc.Big()>0;
01044                                 reg_eip=gate_off;
01045 
01046                                 if (!(gate.Type()&1)) {
01047                                         SETFLAGBIT(IF,false);
01048                                 }
01049                                 SETFLAGBIT(TF,false);
01050                                 SETFLAGBIT(NT,false);
01051                                 SETFLAGBIT(VM,false);
01052                                 LOG(LOG_CPU,LOG_NORMAL)("INT:Gate to %X:%X big %d %s",gate_sel,gate_off,cs_desc.Big(),gate.Type() & 0x8 ? "386" : "286");
01053                                 return;
01054                         }
01055                 case DESC_TASK_GATE:
01056                         CPU_CHECK_COND(!gate.saved.seg.p,
01057                                 "INT:Gate segment not present",
01058                                 EXCEPTION_NP,num*8+2+(type&CPU_INT_SOFTWARE)?0:1)
01059 
01060                         CPU_SwitchTask(gate.GetSelector(),TSwitch_CALL_INT,oldeip);
01061                         if (type & CPU_INT_HAS_ERROR) {
01062                                 //TODO Be sure about this, seems somewhat unclear
01063                                 if (cpu_tss.is386) CPU_Push32(cpu.exception.error);
01064                                 else CPU_Push16(cpu.exception.error);
01065                         }
01066                         return;
01067                 default:
01068                         E_Exit("Illegal descriptor type %X for int %X",(int)gate.Type(),(int)num);
01069                 }
01070                 }
01071                 catch (GuestPageFaultException &pf) {
01072                         LOG_MSG("CPU_Interrupt() interrupted");
01073                         CPU_SetSegGeneral(ss,old_ss);
01074                         reg_esp = old_esp;
01075                         CPU_SetCPL(old_cpl);
01076                         throw;
01077                 }
01078         }
01079         assert(1);
01080         return ; // make compiler happy
01081 }

01223                         n_flags=mem_readw(SegPhys(ss) + (tempesp & cpu.stack.mask));
01224                         n_flags|=(reg_flags & 0xffff0000);
01225                         tempesp=(tempesp&cpu.stack.notmask)|((tempesp+2)&cpu.stack.mask);

01361                         reg_eip=offset&0xffff;

02506 bool CPU_CPUID(void) {
02507         if (CPU_ArchitectureType < CPU_ARCHTYPE_486NEW) return false;
02508         switch (reg_eax) {
02509         case 0: /* Vendor ID String and maximum level? */
02510                 reg_eax=1;  /* Maximum level */ 
02511                 reg_ebx='G' | ('e' << 8) | ('n' << 16) | ('u'<< 24); 
02512                 reg_edx='i' | ('n' << 8) | ('e' << 16) | ('I'<< 24); 
02513                 reg_ecx='n' | ('t' << 8) | ('e' << 16) | ('l'<< 24); 
02514                 break;
02515         case 1: /* get processor type/family/model/stepping and feature flags */
02516                 if ((CPU_ArchitectureType == CPU_ARCHTYPE_486NEW) ||
02517                         (CPU_ArchitectureType == CPU_ARCHTYPE_MIXED)) {
02518                         reg_eax=0x402;          /* intel 486dx */
02519                         reg_ebx=0;                      /* Not Supported */
02520                         reg_ecx=0;                      /* No features */
02521                         reg_edx=enable_fpu?1:0; /* FPU */
02522                 } else if (CPU_ArchitectureType == CPU_ARCHTYPE_PENTIUM) {
02523                         reg_eax=0x513;          /* intel pentium */
02524                         reg_ebx=0;                      /* Not Supported */
02525                         reg_ecx=0;                      /* No features */
02526                         reg_edx=0x00000010|(enable_fpu?1:0);    /* FPU+TimeStamp/RDTSC */
02527                         if (enable_msr) reg_edx |= 0x20; /* ModelSpecific/MSR */
02528             if (enable_cmpxchg8b) reg_edx |= 0x100; /* CMPXCHG8B */
02529                 } else if (CPU_ArchitectureType == CPU_ARCHTYPE_P55CSLOW) {
02530                         reg_eax=0x543;          /* intel pentium mmx (P55C) */
02531                         reg_ebx=0;                      /* Not Supported */
02532                         reg_ecx=0;                      /* No features */
02533                         reg_edx=0x00800010|(enable_fpu?1:0);    /* FPU+TimeStamp/RDTSC+MMX+ModelSpecific/MSR */
02534                         if (enable_msr) reg_edx |= 0x20; /* ModelSpecific/MSR */
02535             if (enable_cmpxchg8b) reg_edx |= 0x100; /* CMPXCHG8B */
02536                 } else if (CPU_ArchitectureType == CPU_ARCHTYPE_PPROSLOW) {
02537                         reg_eax=0x612;          /* intel pentium pro */
02538                         reg_ebx=0;                      /* Not Supported */
02539                         reg_ecx=0;                      /* No features */
02540                         reg_edx=0x00008011;     /* FPU+TimeStamp/RDTSC */
02541                 } else {
02542                         return false;
02543                 }
02544                 break;
02545         default:
02546                 LOG(LOG_CPU,LOG_ERROR)("Unhandled CPUID Function %x",reg_eax);
02547                 reg_eax=0;
02548                 reg_ebx=0;
02549                 reg_ecx=0;
02550                 reg_edx=0;
02551                 break;
02552         }
02553         return true;
